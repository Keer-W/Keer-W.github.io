任务十三

任务描述

  • 参考以下示例代码，补充其中的JavaScript功能，完成一个JavaScript代码的编写
  • 本任务完成的功能为：用户可以在输入框中输入任何内容，点击“确认填写”按钮后，用户输入的内容会显示在“您输入的值是”文字的右边

思路

  • 给按钮绑定事件处理函数handler
  • 在事件处理函数中，将获取到的输入框的值替换在输出框中
  • 可以给回车键也绑定该处理函数

代码实现

	var btn = document.getElementById('button');
	//给按钮绑定的函数
	var handler = function () {
		//获取输入值
		var num = document.getElementById('aqi-input').value;
		//将输出框的值替换为Num
		document.getElementById('aqi-display').textContent = num;
	};
	
	//确认按钮绑定函数handler
	btn.addEventListener('click', handler, false);
	
	//回车键绑定函数handler
	document.getElementById('aqi-input').onkeyup = function(event) {
		if (event.keyCode === 13) {
			handler();
		}
	};

知识回顾

1.JS有三种方法，可以为事件绑定监听函数<br>

  • HTML的on-属性<br>
  • 元素节点的事件属性<br>
  • EventTarget.addEventListener( )<br>

三种方法的区别：<br>

第一种:HTML的on-属性，违反了HTML与JS代码相分离的原则，将两者写在一起，不利于代码分工，不推荐使用<br> 第二种：元素节点的事件属性的缺点在于同一个事件只能定义一个监听函数，也就是说，如果定义两次onclick属性，后一次定义会覆盖前一次。不推荐使用。<br>

第三种：EventTarget.addEventListener是推荐的指定监听函数的方法，优点如下：<br> 1. 同一个事件可以添加多个监听函数<br> 2. 能够指定在哪个阶段（捕获还是冒泡阶段）触发监听函数。<br> 3. 除了这个对象，其他对象也有这个接口，它等于是整个JS的同一监听函数接口。

2. innerHTML与textContent

  • Element.innerHTML属性返回一个字符串，等同于该元素所包含的所有HTML代码。该属性可读写，常用来设置某个节点的内容。如果将innerHTML属性设为空，等于删除所有它包含的所有节点。读取属性值的时候，如果文本节点包含& ，小于号，和大于号，innerHTML属性会将它们转为实体形式&smp;，&lt；，&gt；.如果想得到原文，建议使用element.textContent属性。<br>

  • textContent属性返回当前节点和它的所有节点的文本内容。该属性会自动忽略当前节点内部的HTML标签，返回所有文本内容。该属性可读写，设置该属性的值，会用一个新的文本节点，替换所有原来的子节点。它还有一个好处，就是自动对HTML标签转义。
