未命名

发表于 2018-11-01

写在博客开通前

  大学毕业后也工作一个多月了，是时候总结一下自己的大学生活了，边回忆边写吧，写到哪儿算哪儿。

关于学校

  人生二十三年，自己在高考之前从来未出过远门，高考也差强人意，在被家人浇灭了复读的念头后，到了要挑学校的时候，在这方面家里人完全是有着我的。当时自己对学校和专业的要求很简单：工科+大城市。最后的结果就是来到了天津，作为第一志愿的计算机没有被录取，滑到了第二志愿-电气工程。

  现在回过头来看看当初的选择，无疑是很庆幸的。作为一个来自五线城市的人来说，大城市给自己的改变是方方面面的，无论是格局，还是习惯都让自己发生的很大的改变。再加上毗邻北京这个得天独厚的条件，着实让自己的格局和眼界发生的改变。最重要的是在天津，还遇到了自己的人生伴侣。

关于大学生活

  老实来说，自己并不是一个好学生(不然也不会上普通一个本科)。所以自己的大学并没有什么出彩的地方，没拿过奖学金，没得过奖，挂过科，成绩也一直在中游徘徊，更没有参加过学生会。上课和大部分人也一样，该干的都干过。但总归还是有自己的爱好的，个人认为有很多爱好还是改变了自己，总的来说还是以下三点：

 1.读了除专业课之外的其他各种书。

 2.健身

 3.因为机缘巧合，发现前端技术才是自己的真爱

关于读书



  个人觉得自己买的最划算的东西就是Kindle了，大一499买的基础版，硬是用了四年还能用(良心亚马逊)。但它给自己带来的收益绝对是不能用金钱来衡量的。无论是在上课，地铁还是闲暇时间，它不但使我摆脱了纸质书的厚重，书籍类型也是相当的丰富。这也算是在闲暇时间内自己获取知识的一种有效途径了吧。

关于健身



  自己想健身这件事情最早可以追溯到上高中那会儿，只是限于当时的条件，没能够去健身房，只能放学在家里瞎练。上了大学之后，正式去了健身房，开始总是异常痛苦的，掌握不到要领，相当于还是瞎练。后来看了一本《施瓦辛格健身全书》总算是对健身有了一个基本的了解，再加上自己的坚持，半年就有了效果。

关于前端



  初次接触前端，记得是大三的一门选修课，课程叫啥早已经记不清了。虽然这门课比较水但还是带我走进了前端的大门，从此发现前端的这些东西比专业课有意思多了。闲暇之余也就喜欢去看看相关方面的东西，期间也做过一些小东西，但总归还是小打小闹，没什么拿得出手的东西。现在回头来看看自己当初的前端之路，发现是有很大问题的，简单来说就是两个字–浮躁。现在想想当初的学习，老想着动手，不会就去Google，只要找到解决方法就完事大吉，根本就没有去深入了解其中的原理，也就没有自己的理解。这就使得自己失去了读那些大部头技术书得动力，当自己开始读那些砖头书的时候，时间已经来到了大四。加上自己准备毕业论文，面试，毕业相关等，大四的时间就显得极为紧凑，所以大学期间自己的能力还是较为欠缺的。

  小时候学古诗，诗中说道：读书不觉已春深，一寸光阴一寸金。不是道人来引笑，周情孔思正追寻。大意就是说，不知不觉读书读到春天都已经过去了，时间像金子一样珍贵，若不是被人打断思绪我正在研究学问呢。小时候老师老说时间的重要性，自己那时贪玩也没觉得时间有啥宝贵的，自己天天玩儿多好啊。上了大学虽然觉得时间确实是有限的，但也没到有多宝贵的地步，自己想做的都在不紧不慢的进行着，直到工作了才真真体会到什么叫一寸光阴一寸金，因为已工作你会发现静下心来看本书都成了一件奢侈的事情。上大学的时候，想看啥书就带着kindle去图书馆，教室，在那儿坐一天也没啥事，工作之后随着加班的频繁和工作的劳累，读书的时间可以说是很少了。以后的空闲时间多看看技术书籍，回追基础，静下心来好好提高自己吧。

结束

  啰啰嗦嗦写了这么多，就当是对自己大学生涯的回顾吧，也算是自己博客的开篇。回头看看自己的大学生活，遗憾不少，收获也颇丰。大学生活再美好也回不去了，只能是懵懵懂懂的向前摸索，砥砺前行。

任务十三

任务描述

  • 参考以下示例代码，补充其中的JavaScript功能，完成一个JavaScript代码的编写
  • 本任务完成的功能为：用户可以在输入框中输入任何内容，点击“确认填写”按钮后，用户输入的内容会显示在“您输入的值是”文字的右边

    ##思路##
  • 给按钮绑定事件处理函数handler
  • 在事件处理函数中，将获取到的输入框的值替换在输出框中

  • 可以给回车键也绑定该处理函数
    ##代码实现##
    var btn = document.getElementById(‘button’);
    //给按钮绑定的函数
    var handler = function () {

    //获取输入值
var num = document.getElementById('aqi-input').value;
//将输出框的值替换为Num
document.getElementById('aqi-display').textContent = num;

    };

    //确认按钮绑定函数handler
    btn.addEventListener(‘click’, handler, false);

    //回车键绑定函数handler
    document.getElementById(‘aqi-input’).onkeyup = function(event) {

    if (event.keyCode === 13) {
    handler();
}

    };
    ##知识回顾
    1.JS有三种方法，可以为事件绑定监听函数

  • HTML的on-属性
  • 元素节点的事件属性
  • EventTarget.addEventListener( )

    三种方法的区别：

    第一种:HTML的on-属性，违反了HTML与JS代码相分离的原则，将两者写在一起，不利于代码分工，不推荐使用

    第二种：元素节点的事件属性的缺点在于同一个事件只能定义一个监听函数，也就是说，如果定义两次onclick属性，后一次定义会覆盖前一次。不推荐使用。

    第三种：EventTarget.addEventListener是推荐的指定监听函数的方法，优点如下：
- 同一个事件可以添加多个监听函数<br>
- 能够指定在哪个阶段（捕获还是冒泡阶段）触发监听函数。<br>
- 除了这个对象，其他对象也有这个接口，它等于是整个JS的同一监听函数接口。

2. innerHTML与textContent

  • Element.innerHTML属性返回一个字符串，等同于该元素所包含的所有HTML代码。该属性可读写，常用来设置某个节点的内容。如果将innerHTML属性设为空，等于删除所有它包含的所有节点。读取属性值的时候，如果文本节点包含& ，小于号，和大于号，innerHTML属性会将它们转为实体形式&smp;，&lt；，&gt；.如果想得到原文，建议使用element.textContent属性。

  • textContent属性返回当前节点和它的所有节点的文本内容。该属性会自动忽略当前节点内部的HTML标签，返回所有文本内容。该属性可读写，设置该属性的值，会用一个新的文本节点，替换所有原来的子节点。它还有一个好处，就是自动对HTML标签转义。
